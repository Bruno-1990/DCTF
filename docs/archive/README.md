# Documentação arquivada

Esta pasta contém documentos **obsoletos** ou **apenas históricos** que não são mais necessários no dia a dia do projeto, mas foram preservados para referência.

## Conteúdo

- **correcoes/** – Docs de correções pontuais já aplicadas (ex.: correção erro cliente-id-null, duplicação SITF, etc.).
- **analises/** – Análises antigas (RAG, problemas dispensados, roteiros de implementação, etc.).
- **migracoes/** – Instruções de migração Supabase → MySQL já concluída.

## Quando arquivar

Mover para aqui quando:

- O doc descreve uma correção única já feita.
- O doc é uma análise/roteiro de uma feature já implementada.
- O doc se refere a tecnologia descontinuada (ex.: Supabase) e não é mais consultado.

Não apague sem revisar; em caso de dúvida, manter em `docs/` ou mover para `archive/`.
