# Overview  
O Dashboard Interativo e Inteligente é uma evolução do painel administrativo atual, transformando dados estáticos em insights acionáveis através de visualizações dinâmicas, interatividade inteligente e foco nos dados mais relevantes para tomada de decisão.

**Problema que resolve:**
- Usuários precisam navegar entre múltiplas telas para entender o estado completo do sistema
- Dados importantes ficam escondidos em tabelas estáticas
- Falta de contexto temporal e comparativo para identificar tendências
- Informações de Situação Fiscal (SITF) não estão integradas ao dashboard principal
- Dificuldade em identificar rapidamente problemas críticos e oportunidades

**Público-alvo:**
- Administradores do sistema
- Contadores e gestores fiscais
- Usuários que precisam de visão executiva rápida

**Valor:**
- Visão consolidada e inteligente de todos os dados importantes
- Identificação proativa de problemas e tendências
- Economia de tempo na análise de dados
- Melhor tomada de decisão baseada em dados

# Core Features  

## 1. Visão Executiva Inteligente (Hero Section)
**O que faz:** Cards principais com KPIs mais importantes, com indicadores visuais de tendência (setas para cima/baixo, cores dinâmicas).

**Por que é importante:** Primeira impressão do estado do sistema, permite ação imediata.

**Como funciona:**
- 6 cards principais: Declarações Monitoradas, Saldo em Aberto, Alertas Críticos, Certidões Válidas (SITF), Taxa de Conclusão, Clientes Ativos
- Cada card mostra: valor atual, variação % vs período anterior, indicador visual (verde/vermelho/cinza)
- Cards são clicáveis para drill-down
- Atualização automática a cada 5 minutos (opcional, com toggle)

## 2. Gráfico de Evolução Financeira Interativo
**O que faz:** Gráfico de linha/área mostrando evolução de débito apurado vs saldo a pagar ao longo do tempo.

**Por que é importante:** Identifica tendências financeiras, mostra se situação está melhorando ou piorando.

**Como funciona:**
- Gráfico de área empilhada ou linhas duplas (Recharts)
- Eixo X: períodos (últimos 6-12 meses)
- Eixo Y: valores em R$
- Duas séries: "Débito Apurado" e "Saldo a Pagar"
- Tooltip interativo mostrando valores exatos ao passar mouse
- Filtro de período: 3, 6, 12 meses (botões rápidos)
- Zoom ao clicar e arrastar (opcional)

## 3. Análise de Situação Fiscal (SITF) Integrada
**O que faz:** Seção dedicada mostrando métricas de consultas fiscais, status de certidões e alertas de vencimento.

**Por que é importante:** Dados de SITF são críticos mas estão isolados, precisam estar visíveis no dashboard principal.

**Como funciona:**
- Cards: Total Consultas, Certidões Válidas, Certidões Vencidas, Protocolos Ativos
- Gráfico de pizza: Distribuição de tipos de certidão (Positiva, Negativa, Positiva c/ Efeitos)
- Lista compacta: Top 5 empresas com certidões vencidas ou próximas do vencimento
- Badge de status colorido (verde/amarelo/vermelho) em cada item
- Link direto para aba Situação Fiscal

## 4. Filtros Dinâmicos Inteligentes
**O que faz:** Barra de filtros que permite refinar visualizações sem recarregar página.

**Por que é importante:** Usuários precisam ver dados de períodos específicos, clientes específicos, ou status específicos.

**Como funciona:**
- Filtros: Período (dropdown com presets: Últimos 3/6/12 meses, Mês atual, Trimestre atual), Status (todos/pendente/concluído/erro), Cliente (busca autocomplete)
- Aplicação em tempo real (sem reload)
- Indicador visual de filtros ativos (badge com contador)
- Botão "Limpar filtros" sempre visível
- Filtros persistem na URL (permitem compartilhamento de links)

## 5. Top 10 Clientes com Maior Saldo Pendente
**O que faz:** Gráfico de barras horizontal mostrando os 10 clientes com maior saldo pendente.

**Por que é importante:** Identifica rapidamente onde está concentrado o risco financeiro.

**Como funciona:**
- Gráfico de barras horizontal (Recharts)
- Eixo Y: Nome/Razão Social do cliente (truncado se muito longo)
- Eixo X: Valor em R$ (formatado)
- Cores: gradiente de vermelho claro a vermelho escuro (maior = mais escuro)
- Tooltip: mostra CNPJ, saldo exato, número de declarações pendentes
- Clique na barra: navega para página DCTF filtrada por aquele CNPJ

## 6. Comparativo Período Atual vs Anterior
**O que faz:** Cards comparativos mostrando variação entre período atual e anterior.

**Por que é importante:** Contexto é essencial - saber se números estão melhorando ou piorando.

**Como funciona:**
- Cards lado a lado: "Este Mês" vs "Mês Anterior"
- Métricas: Total Declarações, Saldo Pendente, Taxa Conclusão, Alertas
- Indicador de variação: % e seta (verde = melhorou, vermelho = piorou)
- Gráfico pequeno de barras lado a lado para comparação visual

## 7. Alertas Inteligentes Contextualizados
**O que faz:** Seção de alertas melhorada com agrupamento inteligente e ações rápidas.

**Por que é importante:** Alertas precisam ser acionáveis, não apenas informativos.

**Como funciona:**
- Mantém agrupamento por tipo
- Adiciona: badge de contador em cada grupo
- Botão "Resolver" em cada alerta (marca como visto/resolvido)
- Filtro rápido: "Mostrar apenas críticos"
- Indicador de novos alertas desde última visita (badge vermelho)
- Link direto para ação corretiva quando aplicável

# User Experience  

## User Personas

**Persona 1: Administrador do Sistema**
- Precisa: visão geral rápida, identificar problemas críticos, monitorar saúde do sistema
- Uso: diário, 5-10 minutos por sessão
- Expectativa: dados atualizados, ações rápidas, sem poluição visual

**Persona 2: Contador/Gestor Fiscal**
- Precisa: entender situação financeira, identificar clientes com problemas, acompanhar prazos
- Uso: várias vezes ao dia, 2-5 minutos por sessão
- Expectativa: foco em dados financeiros e prazos, fácil navegação para detalhes

## Key User Flows

**Fluxo 1: Verificação Rápida Diária (2 minutos)**
1. Usuário acessa dashboard
2. Vê cards principais (identifica problemas imediatamente)
3. Verifica alertas críticos (se houver, clica para ver detalhes)
4. Verifica gráfico financeiro (vê tendência)
5. Se tudo OK, fecha dashboard

**Fluxo 2: Análise de Cliente Específico (5 minutos)**
1. Usuário acessa dashboard
2. Usa filtro de cliente (busca por CNPJ ou nome)
3. Dashboard atualiza mostrando apenas dados daquele cliente
4. Analisa gráficos e métricas filtradas
5. Clica em item específico para drill-down
6. Navega para página de detalhes

## UI/UX Considerations

**Design Principles:**
- Menos é mais: máximo 3-4 seções visíveis por vez (scroll suave)
- Hierarquia visual clara: informações mais importantes em destaque
- Cores semânticas: verde (bom), amarelo (atenção), vermelho (problema)
- Feedback imediato: loading states, animações suaves, transições
- Responsivo: funciona bem em desktop e tablet (mobile: versão simplificada)

**Interatividade:**
- Hover states em todos elementos clicáveis
- Tooltips informativos sem poluir
- Animações suaves (fade, slide) para mudanças de estado
- Loading skeletons (não spinners) durante carregamento
- Transições entre estados de filtro

# Logical Dependency Chain

## Ordem de Desenvolvimento (Foundation First)

**1. Backend: Enhanced Dashboard Service (Fundação)**
- Precisa ser feito primeiro pois frontend depende dos dados
- Expandir `DashboardMetricsService` existente
- Criar novos serviços (SITF, Comparativos)
- Criar endpoints de API
- **Resultado:** API funcional retornando dados estruturados

**2. Frontend: Hero Section (Visibilidade Rápida)**
- Primeiro componente visível
- Usa dados do backend criado
- Validação de integração backend-frontend
- **Resultado:** Usuário vê dados importantes imediatamente

**3. Frontend: Gráfico Financeiro (Primeira Visualização)**
- Depende de Hero Section (mesma fonte de dados)
- Valida biblioteca de gráficos (Recharts)
- **Resultado:** Primeira visualização interativa funcionando

**4. Frontend: Top Clientes (Segunda Visualização)**
- Independente do gráfico financeiro
- Valida padrão de criação de gráficos
- **Resultado:** Segunda visualização, padrão estabelecido

**5. Backend + Frontend: Integração SITF**
- Backend: Criar `SitfDashboardService`
- Frontend: Criar `SitfMetricsSection`
- **Resultado:** Dados SITF integrados ao dashboard

**6. Frontend: Filtros Dinâmicos**
- Depende de todos componentes anteriores (aplica filtros neles)
- Mais complexo, mas essencial para usabilidade
- **Resultado:** Dashboard interativo e personalizável

**7. Frontend: Comparativos e Melhorias**
- Depende de dados estáveis
- Polish e melhorias visuais
- **Resultado:** Dashboard completo e polido

**8. Frontend: Alertas e Exportação**
- Features avançadas
- Depende de tudo funcionando
- **Resultado:** Dashboard completo com features avançadas

# Risks and Mitigations  

## Technical Challenges

**Risco 1: Performance com muitos dados**
- **Mitigação:** 
  - Implementar paginação/carregamento lazy
  - Cache agressivo (5 minutos)
  - Queries otimizadas com índices
  - Limitar dados por padrão (últimos 6 meses)

**Risco 2: Complexidade de filtros**
- **Mitigação:**
  - Cache por combinação de filtros
  - Debounce em mudanças de filtro
  - Limitar filtros simultâneos inicialmente
  - Queries parametrizadas eficientes

**Risco 3: Integração SITF (dados em 2 bancos)**
- **Mitigação:**
  - Criar serviço unificado que agrega de ambos
  - Cache de dados SITF (menos frequente)
  - Tratamento de erros se um banco falhar

## MVP Definition

**MVP Mínimo Viável:**
- Hero Section com 4-6 cards principais
- Gráfico de evolução financeira básico
- Integração SITF básica (cards apenas)
- Filtros básicos (período apenas)
